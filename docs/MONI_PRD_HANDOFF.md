# Moni（家庭/个人收支）— 产品与技术交接文档

> 供新 Cloud Agent（仓库 `zephyrkeaton/moni_android`）直接阅读。  
> 本文汇总用户与前一轮 Agent 约 12 小时内的定案，可视为 **V1+ 分期 PRD 骨架**。  
> 生成日期：2026-07-24（UTC）

---

## 1. 项目定位

- **名称暂定**：Moni（仓库：`zephyrkeaton/moni_android`）
- **平台**：先做 **Android**，随后做 **iOS**（策略 C，已冻结）
- **核心卖点**：隐私优先、**默认可完全离线**；本地记账；家庭账本通过 **面对面**（蓝牙为主，NFC 可后置）同步，不做传统云端实时多端同步（网盘备份另计）
- **目标用户**：个人 + 家庭（夫妻/孩子）；同时支持多账本场景（打工、学生、老板、副业、家财等）

---

## 2. 已冻结的关键决策（务必遵守）

| # | 决策 |
|---|------|
| 1 | **本机仅 1 个用户**（PIN/密码 + 生物识别解锁 App），不做多用户登录 |
| 2 | **每用户最多 10 个账本**；创建用户后引导创建账本；可同时存在多个账本 |
| 3 | **双端确定要做**：Android V1 先行 → 之后 iOS；**备份/同步包从第一天起跨平台中立** |
| 4 | 开发分期按下文 V1 → V1.5 → V2 → V2.5 → iOS |
| 5 | OCR：MVP 不做；后期「辅助预填，用户确认后入账」；先中文再英/日 |
| 6 | 报销/退款：V1 **只留数据字段**，UI 放后续 |
| 7 | 可后置：银行自动导入、投资持仓、云实时同步等 **暂不做** |

## 3. 原始需求与修订对照

### 3.1 认证与本地用户

- 不需要联网即可运行
- 首次运行创建本地用户信息与登录认证信息
- 支持系统指纹 / 面部识别；通过认证后才能进入
- **定案**：本机单用户保险库；生物识别解锁本地凭证；须有 PIN/密码兜底

### 3.2 日历与记账

- 有日历显示界面
- 任意一天可创建收入/支出
- 字段至少包括：分类、名称、金额，以及其它合理字段
- **已追加**：「为谁花的」字段
- **已同意**：支付方式/支付渠道；标签；成员相关字段

### 3.3 汇总

- 整月：总收入、总支出、合计（结余）
- 周：同上
- **周首日**：用户可选 **周日** 或 **周一**
- **周首日变更**：明确警告，**从下一周开始生效**；只影响周汇总与周预算结转，**不影响**月/季/年
- **因此不需要全历史重算周结转链**（工作量可控）

### 3.4 OCR（后置）

- 调用摄像头读购物小票 → 识别文字 → 预填新建支出页 → 用户修改并确认后入账
- 目标语言：中文、英文、日文
- **定案**：先 MVP，再 OCR 增强；识别仅辅助预填

### 3.5 多语言 UI

- 界面可在 **中文 / 日文 / 英文** 间切换
- **切换语言时**：提示用户即将发生金额符号、金额格式、日期格式等变化；建议「确认后生效」并可预览示例
- **货币**：**每个账本独立设定货币**，不跟随手机系统

### 3.6 预算

- 可设定某月或某周的支出预算，实时对比预算 vs 实际
- **追加**：季度预算、年预算
- **超预算**：醒目显示；并提供 **无障碍 / 色弱** 友好选项（不只靠红绿）
- **结转**：结余与超支 **都结转**；结转到 **下一个同类型周期**（周→周，月→月，季→季，年→年）

### 3.7 账本（非多登录用户）

- 允许创建多个独立账本（上限 10）
- 场景模板示例：老板、打工人、学生、副业、家财万贯（家里有钱不用工作）等；**空白模板**也要有
- 不同模式给出不同角色场景所需的 **预设特点**（分类、预算建议等）
- 在选择账本层：入口进入 **横向对比所有账本** 的统计页

### 3.8 家庭共享（面对面）

- 家庭成员可共享账本，**仅限物理世界面对面**，用 NFC 或蓝牙传文件
- **可行性**：可行；建议 V2 以 **蓝牙（或系统分享）传加密账本包** 为主；NFC 作快捷入口可更后
- **不是**实时云同步，文案应写「当面同步」

### 3.9 成员分色与统计

- 典型夫妻场景：丈夫收支用男性色、妻子用女性色；两端一致按 **角色** 上色，统一统计
- **追加角色**：孩子1、孩子2、孩子3 等
- **孩子可作为记账人**；年龄太小由父母代操作，旨在培养记账习惯
  - 建议字段：`createdBy`/归属成员 = 孩子；`recordedBy` = 实际操作的父母
- 主题：**5 套**；饱和度约 **40**（0–100 标度）；配合图标/文字，服务色弱

### 3.10 公共支出与分摊

- 水电房租等公共支出：录入时可选择某一方负担，或双方按比例
- 周期自动添加的公共支出：**某一期实例**可单独改「本次谁付 / 本次比例」，不影响模板
- **分摊比例**：默认 **50/50**；可改但 **只能整十**（0,10,20,…,100），含 0/100；另一方 = 100 − 该值

### 3.11 共享编辑与解散

- 当面合并后：希望 **只能修改自己的**；改对方内容属于现实沟通；对方改完再同步
- **允许修改但留修改记录（audit log）** — 可作为高级能力；主路径仍是「只改自己的」
- **解散共享**含义：解除「双方持续同步同一共享账本」的关系；之后不再自动合并
- **成员改名 / 离婚式退出**：保留以往 **完整历史副本**；成员显示名可改，历史用稳定成员 ID

### 3.12 同步冲突策略（定案）

- 默认可改自己创建的记录；对方记录只读（或改则强确认 + 日志）
- 物理沟通 → 对方修改 → 再当面同步
- 极端同笔双改：用 `updatedAt` + 日志，同步时提示冲突版本

### 3.13 备份与恢复

- 备份到手机本地，以及 Google 云、Box、Dropbox、百度云、阿里云等
- 可从备份恢复
- **分期**：V1 本地备份必做；Google Drive 放较后；百度/阿里更后
- 备份包格式：**跨平台中立**（服务后续 iOS）

### 3.14 隐私

- 写入 App 内「关于此 App」类页面，为上架 Google Play 等做准备

### 3.15 跨账本对比与货币

- 对比：简单汇总所有账本；**排除内部转账**
- 共享账本与个人账本 **不应双记**：一笔钱只入一个账本（用户选择入哪里）— **用户理解正确，按此实现**
- 货币不同时：弹窗让用户 **手输汇率**；显示两种货币与两个金额框（原币 / 换算）；不内置在线汇率

### 3.16 数据模型类型

- 支持：**支出、收入、转账、退款、报销**
- V1 UI：转账可视需要做最小能力；**退款/报销入口后置**，字段先留

## 4. 预算规则（开发按此实现）

### 4.1 周期定义

| 周期 | 定义 |
|------|------|
| 周 | 用户选周日或周一为周首；设置变更从 **下一周** 生效 |
| 月 | 自然月 1 日–月末 |
| 季 | 自然季：1–3 / 4–6 / 7–9 / 10–12 |
| 年 | **二选一（每账本可配）**：① 自然年 1/1–12/31；② **自定义年起始日**（如 4/1 → 至下一年同日前一天） |
| 年起始日变更 | 警告后从 **下一个年周期** 生效（对齐周首日策略） |

- 周/月/季/年预算 **彼此独立**，不做自动拆分互相同步
- 自定义年 **不影响** 自然季定义

### 4.2 结转公式

对每个周期类型独立：

```
结转额 = 本期预算 - 本期实际支出   // 可正可负
下期可用 = 下期基础预算 + 结转额   // 若用户选择「允许结转吞掉下期」
```

用户可选择：

1. **结转并允许下期预算被完全吞掉**（可用为 0 或负，需醒目提示）
2. **一键清空结转，下个周期按基础预算重新开始**

### 4.3 超预算展示

- 醒目字体/样式 + 色弱友好（图标、纹样、文字，不只靠颜色）

## 5. 周期自动记账（定案）

- 房租、工资、订阅等金额相对固定的收支：建议用户设为周期性
- **提醒用户确认后**，进入该周期时自动入账
- 金额变化时确认：
  - **仅本次**变化，或
  - **从本次起**以后都变（改模板）
- **历史实例不变**

---

## 6. 账本模板（预设方向）

| 模式 | 预设侧重 |
|------|----------|
| 打工人 | 工资收入、通勤餐饮等生活分类、月预算 |
| 学生 | 生活费/学费、周预算更有用 |
| 老板/经营 | 经营收支、季度/年预算、报销与对公支付方式（字段预留） |
| 副业 | 与主业分离的收入成本分类 |
| 家财/家庭 | 共享成员、公共分摊、房租水电周期、家庭向 |
| 空白 | 最少预设，全自定义 |

## 7. 同意纳入的能力清单

### 7.1 高价值（用户明确同意）

1. 分类体系（支出/收入，可自定义）
2. 支付方式 / 支付渠道
3. 图表与趋势
4. 搜索与筛选
5. 周期性收支（见 §5）
6. 本地备份与恢复（跨平台格式）
7. 成员 / 标签；「为谁花的」
8. 预算分层（总预算 + 后续可分类预算）

### 7.2 中价值（用户赞成）

9. 仪表盘首页  
10. 收据/附件（小票图归档）  
11. 退款 / 报销 / 转账（模型支持；UI 分期）  
12. 多币种（每账本货币 + 对比手输汇率）  
13. 提醒（记账、预算阈值等）  
14. 回收站  

### 7.3 家庭向（用户赞成）

15. 共同账本 vs 个人账本  
16. 分摊规则（整十比例）  
17. 月度家庭报告（可后置于图表阶段）  

### 7.4 明确暂不做

- 云实时同步、银行自动导入、投资持仓、贷款计划、发票验真、AI 自动分类等

## 8. 开发分期（已确认按此开展）

| 阶段 | 内容 |
|------|------|
| **V1 Android** | 单用户；≤10 账本与模板；日历记一笔（分类/名称/金额/为谁花的/标签/支付方式）；月/周汇总；月/周预算+结转选项+超支提示；语言切换+格式变更提示；本地加密备份/恢复（**中立格式**）；隐私/关于页；无障碍基础；数据模型含转账/退款/报销字段；PIN+生物识别 |
| **V1.5** | 季/年预算（含自定义年起点）；图表；仪表盘；搜索筛选；回收站；提醒；周期确认入账；退款/报销 UI 可开始露 |
| **V2** | 共享账本；成员（夫/妻/孩1–3）；分摊；5 套分色主题（饱和度≈40）；蓝牙当面同步；只改自己的+修改记录；解散共享留完整副本 |
| **V2.5** | Google Drive 备份；OCR 预填（先中文） |
| **更后** | 其它网盘；NFC 快捷；英日 OCR 增强等 |
| **iOS 客户端** | Android 主路径稳定后启动；复用中立备份/同步包与业务规则语义；UI/生物识别/蓝牙/上架按平台重做 |

## 9. 技术约束与建议（给实现 Agent）

1. **仓库**：`https://github.com/zephyrkeaton/moni_android`（用户已新建；可能为 Private，需 Cursor GitHub App 已授权）
2. **前一轮 Agent 所在仓库**是 `zephyrkeaton/vava8`，**无法写入 moni_android**；本文件为交接用，请在 **moni_android** 落代码
3. **跨平台数据**：定义版本化导出/同步包（建议加密容器 + 清单 + 校验和）；字段文档化，供 iOS 复用
4. **本地存储**：加密（如 SQLCipher / EncryptedSharedPreferences 等按栈选择）
5. **生物识别**：Android BiometricPrompt；须 PIN 兜底
6. **当面同步**：先蓝牙/系统分享加密包 + 合并算法；NFC 后置
7. **一笔只进一个账本**；内部转账不计入收支汇总与跨账本对比
8. **周首日 / 年起点变更**：下期生效，避免全历史重算
9. **iOS**：不与 Android V1 并行开发，但格式与规则层从 V1 就要中立
